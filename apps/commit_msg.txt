feat(calendar): implement persistent selection mode and triple-write sync protocol

Refactored the calendar interaction model to improve stability and prevent 
accidental temporal mutations. Significant architectural improvements to 
the synchronization lifecycle.

Key Changes:
- Interaction: Implemented friction-based 'Long Press' to activate event 
  selection handles. Handles now persist until background tap or commit.
- Logic: Unified drag-and-drop and resize operations under the 
  'onDragSelectedEventEnd' callback with defensive timestamp normalization.
- Sync: Implemented 'Triple-Write' protocol in executeEventUpdate:
    1. Authoritative Cloud mutation (Convex).
    2. Local instance synchronization (SQLite).
    3. Hardware trigger refresh (Native Alarms).
- UX: Integrated ConfirmationModal for staging temporal shifts and 
  added overlap error handling for schedule collisions.
- Docs: Added comprehensive architectural comments detailing design 
  patterns (Headless Range Management, Persistence layers).

Fixes identifier resolution between backend (_id) and UI-mapped (id) instances.
