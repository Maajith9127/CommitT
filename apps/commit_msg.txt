fix(security): harden SecurityShield against "Ghost ADB" attack vector

- Added `isUsbDebuggingEnabled` native function to `BlockerModule` to query `Settings.Global.ADB_ENABLED` directly from the Android system database.
- Integrated a 4th security layer into `SecurityShield` that detects active USB debugging even if the Developer Options menu has been hidden via shell commands (e.g., `development_settings_enabled 0`).
- Updated the security violation UI to explicitly list USB Debugging as a restricted configuration.

This neutralizes the "Ghost ADB" exploit where a user could keep the ADB bridge operational for remote database manipulation while bypassing standard menu-based detection.
