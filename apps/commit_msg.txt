chore(sync): resolve native deadlocks and "Loop of Doom" sync cycles

This commit hardens the native synchronization engine to prevent deadlocks
and redundant reconciliation cycles, particularly on Android ROMs where 
JS-side swipe-killing does not immediately terminate native threads.

Key Infrastructure Changes:
- SyncLock V2: Implemented a 30s/45s Operation Timeout Guard. If a Convex 
  query or SQLite transaction hangs, the lock is now forcibly released 
  to prevent permanent system deadlocks.
- Manual Resync Awareness: Added an `isManualResyncActive` flag to 
  coordinate between the Profile resync saga and the background engine.
- HydrationSync Circuit Breaker: Limits consecutive "Amnesia" detections 
  to 3 attempts, preventing infinite log-flooding loops.
- Cold Boot Reset: Added an explicit SyncLock reset on initial mount 
  to clear "zombie" locks surviving from previous process lifetimes.
- SQLite Integrity: Hardened the "Nuke & Pave" logic with VACUUM and 
  TRUNCATE to eliminate fragmented WAL boundaries causing JVM OOMs.

Resolves persistent sync hangs on Lenovo K12 Note.
