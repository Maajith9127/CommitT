feat(sync): implement hardened 'Fetch-Before-Wipe' resync protocol

This commit finalizes the security and integrity strategy for manual 
data resynchronization in the Profile screen, eliminating offline 
vulnerabilities and improving infrastructure stability.

Key Improvements:
- Fetch-Before-Wipe: Re-architected the resync flow to download entire 
  cloud snapshots to RAM before initiating the local SQLite wipe. This 
  neutralizes 'offline amnesia' exploits where users could attempt to 
  clear local blocks while disconnected.
- Connectivity Hard-Gate: Added a mandatory Stage 0 internet check 
  (Google ping) to verify reachability before any data operations.
- Infrastructure Documentation: Integrated high-impact engineering 
  rationale (🛡️, 🏗️, 🔄) explaining the security, integrity, and 
  coordination logic for the resync engine.
- Sequential Stability: Optimized the execution order to ensure 
  compatibility with SQLite's PRAGMA and VACUUM commands, preventing 
  lock contention and Android OOM crashes.
- Semaphore Coordination: Refined the 'isManualResyncActive' logic 
  to cleanly silence background sync triggers during the operation.

Ensures the 'Imperial Override' is both exploit-proof and 100% 
reliable across different Android hardware variants.
