feat(navigation): implement native ripples and map contextual header icons

This commit focuses on standardizing interactive navigation feedback and polishing layout metadata across the main application views.

Navigation & Interactions:
- Refactored `BottomBar.tsx` to use React Native's `Pressable` API instead of `TouchableOpacity`.
- Implemented a high-visibility, native `android_ripple` (opacity 0.4, borderless) on bottom tab icons for immediate tactile feedback.

Header Contextualization:
- Updated the `getHeaderConfig` in `(main)/_layout.tsx` to render screen-specific Material Icons.
- The `Presets` tab now displays the `tune` icon.
- The `Profile` tab now displays the `account-outline` icon.

Layout Refinements:
- Tightened vertical rhythm in `profile.tsx` by removing excessive `mb-4` margin layers around the 'Account' and 'Customize' sections.
- Reduced the primary identity block margin (`mb-10` -> `mb-6`), successfully anchoring the scrollable content closer to the top avatar circle.
