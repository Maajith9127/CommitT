fix(android): resolve native crash on Alarm trigger for Android 8-12

Summary:
Surgically patched the scheduler-module to address two critical failure points causing instant process crashes at alarm trigger time on specific OEM devices (e.g., Lenovo).

Key Changes:
- build.gradle: Added missing 'androidx.appcompat:appcompat' dependency to resolve Theme.AppCompat resource errors.
- AlarmActivity.kt: Hardened the VibrationAttributes API guard by correctly targeting API 33 (Tiramisu). This prevents NoClassDefFoundError on devices running Android 8 through 12.

Impact:
Extends stable hardware alarm support to all devices from API 26 (Oreo) upwards, ensuring the "Forward-Heal" overlay fires reliably across different Android versions.
