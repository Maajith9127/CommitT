feat: implement sovereign grading engine and style-aware verification

### The Problem
The verification runner was performing a naive binary check on the root instance status. It failed to account for complex habit styles like "Stay Throughout" (which requires ongoing presence) or "Just Show Up" (which requires a single check-in within a window), relying instead on external processes to update the status before the window closed.

### The Fix
This commit introduces a "Sovereign Grading Engine" that performs deep telemetry analysis of checkpoints to determine fulfillment independent of external state updates:

- **Modular Grading Logic**: Extracted verification rules into `evaluateGradingVerdict()`, separating the "Judgment" (logic) from the "Enforcement" (mutations).
- **Style-Aware Telemetry**: 
    - **Stay Throughout**: Now analyzes checkpoint arrays and counts 'missed' windows. It enforces a threshold based on `max_missed_checkins`, allowing for defined tolerance in habit adherence.
    - **Just Show Up**: Evaluates checkpoints for a terminal 'verified' state across at least one window, ensuring point-in-time commitments are honored.
- **Production-Grade Observability**: Standardized log formats and added extensive JSDoc documentation to detail the "Heart-First Handoff" and "Redemption Arc" architectures.
- **Emoji-Free Formalism**: Cleaned up the log output and code comments to match high-reliability production standards.

### Technical Changes
- **runner.ts**: Implemented `evaluateGradingVerdict()`; unified Phase 2 grading to use style-specific rules; refined Phase 3 enforcement logs.
- **scheduler.ts**: Enhanced JSDoc to document the 'Temporal Firewall' and 'Surgical Cleanup' patterns.
- **update.ts**: Updated JSDoc to reflect the 'Reactive Reconciliation' requirement for UI sync.
