feat(sync): implement 100-tap 'Discipline Gate' for manual resync

This commit introduces a high-friction interaction layer to the Full 
Resync protocol, designed to eliminate impulsive bypass attempts and 
reinforce the user's commitment to the app's discipline model.

Key Changes:
- Manual Friction Gate: Implemented a mandatory 100-tap requirement to 
  trigger the 'Imperial Override' resynchronization. 
- Real-time Countdown UI: Designed a dynamic title system that shows 
  exactly how many more taps are required (e.g., "Tap 48 more times").
- Visual Escalation: Added a 'Danger State' where the interaction 
  button turns red after 80 taps, signaling the final threshold before 
  the local database wipe.
- Reset Logic: Enforced strict state management—if the user closes the 
  modal before the 100th tap, the progress is wiped, ensuring only 
  100% deliberate actions succeed.
- Engineering Rationale: Added production-level documentation 
  explaining the psychological friction model used to protect the 
  app's integrity.

Transforms a critical recovery tool into a deliberate, labor-intensive 
process that matches the 'Hard Mode' ethos of CommitT.
