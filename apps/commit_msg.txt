feat(penalty): architect end-to-end penalty integration with deferred upload and backend URL resolution

This commit establishes the complete accountability contract pipeline, bridging the native 
frontend draft state with Convex Storage and the backend service layer.

### 🏗️ Technical Architecture & Rationale:

1.  **Deferred Asset Upload Strategy**: 
    Implemented a middleware layer in the 'useCommitTask' orchestrator. Photo assets are kept 
    locally as 'file://' URIs during the configuration phase to save bandwidth. Uploads to 
    Convex Storage are only triggered upon final "Commit," ensuring zero wasted storage for 
    abandoned drafts.

2.  **Stateless Frontend / Stateful Backend**:
    Zustand now consistently tracks 'photoUrl'. On submission, the orchestrator interceptor 
    swaps local URIs for permanent 'storageId' references. The backend then performs 
    synchronous URL resolution, storing BOTH the 'storageId' (for DB integrity) and 
    the 'photoUrl' (for instant UI rendering on reload).

3.  **Centralized File Registry**:
    Integrated 'api.files.record' into the upload pipeline. Every penalty asset is now 
    registered in the 'files' table with a 'penalty_photo' tag, enabling robust ownership 
    tracking, easier garbage collection, and user-level asset audits.
