refactor(ui): establish centralized Saffron & Obsidian design system

This commit transitions the application from scattered, hardcoded Tailwind utility classes to a high-production, centralized Design System (Theme Tokens), laying the groundwork for the new "Saffron Command Center" aesthetic.

Tooling Fixes:
- fix(scripts): Updated the `connect-phone.ps1` script regex. It now uses a strict `Where-Object` filter to explicitly ignore "Not shared" states, resolving the false-positive WSL attachment bug.

UI Architecture & Theming:
- feat(theme): Created `constants/theme.ts` as the single source of truth for the app's aesthetic, introducing the new primary Saffron color (`#C6613F`) and deep Obsidian backgrounds.
- refactor(buttons): Overhauled all shared button components (`PrimaryButton`, `SecondaryButton`, `AddButton`, `LinkButton`) in `button.tsx` to dynamically pull from `THEME.colors`.
- refactor(typography): Updated `text.tsx` components to use `THEME.colors.textMain` and `textMuted` for seamless future-proofing.
- style(VerificationCard): Refactored `VerificationCard.tsx` to utilize the new surface tokens. Applied the vibrant Saffron color to the dynamic status border and shield icon to distinctly differentiate the app's aesthetic from generic Play Store blockers.
