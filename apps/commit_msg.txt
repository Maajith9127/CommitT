ux: implement full-lifecycle loading feedback for asynchronous operations

[Problem]
High-latency transactions (Cloud + Cache + Hardware sync) felt "stuck" due 
to lack of visual feedback. Confirmation modals disappeared instantly, 
leaving a 200ms-400ms gap where the app appeared frozen while background 
sync was still active.

[Solution]
Integrated a unified loading state architecture across all critical 
Commitment/Instance workflows.

Key Changes:
- UI: Added 'isLoading' prop to 'ConfirmationModal' with integrated 
  ActivityIndicator swap-logic to prevent layout shift.
- Safety: Implemented strict button-disabling during loading to prevent 
  double-submission and race conditions in SQLite transactions.
- Schedules: Added feedback for drag-and-drop temporal shifts and 
  conflict-resolution ack.
- Creation: Locked the 'FinalScreen' modal during the "Triple-Write" 
  protocol (Convex -> SQLite -> Android Native) to ensure full consistency.
- Detail View: Refined instance deletion to show active progress within 
  the event modal.

[Impact]
- Eliminates "App Freeze" perception by providing 60fps visual feedback.
- Prevents database corruption from duplicate concurrent mutations.
- Improves perceived performance and trust through deterministic states.
