fix(sync): implement background resilience and SQLite self-healing

Resolved critical 'ERR_ACCESS_CLOSED_RESOURCE' failures and Convex WebSocket 
timeouts observed in production release builds caused by aggressive Android 
resource reclamation.

Resilience Architecture:
- Implemented `AppState` foreground detection to automatically trigger state 
  reconciliation whenever the user returns to the application.
- Engineered a robust SQLite health-check loop that retries connection probes 
  up to 3 times with a 1.5s delay to survive OS-level handle invalidation.
- Introduced a 10-second debounce guard and a mutex-based execution lock 
  (`isSyncingRef`) to prevent redundant sync cycles and race conditions.
- Enhanced version mismatch recovery logic to force a clean handshake 
  immediately following backend redeployments.

This architecture ensures the local-cloud bridge remains consistent even after 
prolonged backgrounding or OS-level memory pressure.
