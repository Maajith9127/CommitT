fix(native): reroute EventDetailModal deletion to instance-scoped API

Fixed a critical bug where attempting to delete an individual task occurrence 
from the EventDetailModal triggered the parent 'commitments/delete' API instead 
of the granular 'instances/delete' API. This caused Convex validation errors 
due to Table ID mismatches (taskInstances vs tasks).

- Updated useEventDetail hook to destructure and invoke 'deleteInstance'
- Added architectural documentation to project_context.md clarifying deletion scopes
