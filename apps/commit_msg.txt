feat(accountability): implement sovereign enforcement and high-fidelity deterrents

This commit hardens the core accountability engine by ensuring backend 
execution resilience and implementing high-fidelity deterrents in the UI.

### ⛓️ Sovereign Enforcement & Dispatching
- **Decoupled Worker:** Refactored `firePenalty` worker to operate 
  independently of parent task records, relying entirely on the frozen 
  `instance.penalty` snapshot to ensure accountability for manual edits.
- **Scalable Dispatcher:** Created a centralized [dispatcher.ts](cci:7://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/packages/backend/convex/core/penalties/dispatcher.ts:0:0-0:0) and 
  modular executors (e.g., [send_email.ts](cci:7://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/packages/backend/convex/core/penalties/executors/send_email.ts:0:0-0:0)) to route various penalty 
  types (Email, Photo, Money) with a standardized [PenaltyResult](cci:2://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/packages/backend/convex/core/penalties/dispatcher.ts:12:0-15:2) flow.
- **Proactive Defusal:** Updated the server-side janitor ([service.ts](cci:7://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/packages/backend/convex/core/instances/service.ts:0:0-0:0)) 
  to proactively cancel scheduled penalty jobs during task deletion or 
  updates, targeting all "In-Flight" states (`pending`, `proceeding`, `waiver_active`).

### 📸 High-Fidelity Accountability (UI)
- **Threat Previews:** Enhanced [VerificationStatusCircle](cci:1://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/apps/native/components/ui/commits/VerificationStatusCircle.tsx:67:0-180:1) to support 
  thumbnails. Penalties using "Embarrassing Photo" now display a circular 
  preview of the actually uploaded stake, serving as a powerful visual reminder.
- **Dynamic Content Routing:** Refactored `EventDetailConditions` to display 
  specific destination details (e.g., "Photo will be sent to [email]") and 
  dynamic waiver counts (e.g., "Solve 5 CAPTCHAs") via a live subscription.
- **Visual Urgency:** Standardized all penalty cards to use Commit 
  "Failure Red" (#FF3B30) for consistent psychological impact.
- **Reactive Status Mapping:** Integrated real-time instance status into 
  the condition rows, providing immediate feedback for active waivers, 
  successful bypasses, and fired penalties.
