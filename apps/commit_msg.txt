feat(native/ui): overhaul Profile and Configuration Menus

- Migrated `profile.tsx` to utilize `SettingsToggleCard` for a sleek, list-based 
  navigation layout mimicking native iOS/Android settings panels.
- Dynamically hooked `profile.tsx` top-section to the active `authClient` Google 
  Sign-In session to natively render the user's fetched avatar, name, and email.
- Upgraded the `SettingsToggleCard` engine to natively support `@expo/vector-icons`, 
  automatically rendering material icons beside headings when provided in the config array.
- Injected contextual material icons into all 10+ configuration rows in `final.tsx` 
  (Just Show Up, Grace Period, Alarms, Waivers).
- Optimized layout sizing in `final.tsx` by introducing compact value strings 
  (e.g., "-30m", "5m", "12h") instead of verbose text to naturally prevent UI 
  collisions on smaller mobile screens.
