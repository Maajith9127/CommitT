style(ui): global hardening of (create-commit) flow & shared status components

Complete elimination of legacy hex codes (#4FA0FF, #9CA3AF) and hardcoded Tailwind utility colors across all configuration modals and picker screens. This migration centralizes the visual authority to the Saffron & Obsidian (THEME) token architecture.

Technical Changes:
- Shared Primitives: Updated `VerificationStatusCircle` and `ActionMenu` to use dynamic `surfaceLight` and `surfaceElevated` tokens, restoring tactical depth and high-contrast borders (1.5px).
- Blocklist Orchestration: Refactored `choose.tsx` and its internal module tree (`TabsBar`, `TopBar`, `InlineAddBar`, `SelectableListItem`) to map all interactive states and layouts to the centralized design system.
- Preset Pickers: Tokenized `DigitalPresetPickerModal` and `RulePresetPickerModal` to standardize background-level surfaces, empty states, and destructive action color mappings.
- Typography: Synchronized `HeaderTitle` and `AuthTitle` usage across modals to ensure consistent font scaling and legibility on high-density displays.
- Asset Integration: Injected dynamic theme color parsing into Google Maps Static API requests within the location selection logic.

Impact:
Ensures 100% aesthetic consistency across the commitment creation lifecycle and facilitates instant global rebranding via the `theme.ts` authority.
