feat(ui): implement dynamic penalty and waiver details in event modal

This commit enhances the event detail view by replacing static placeholders 
with live, configuration-driven details for both penalties and waivers.

### 🎯 Dynamic Accountability UI
- **Live Waiver Details:** Refactored [WaiverSection](cci:1://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/apps/native/components/ui/modal/EventDetailConditions.tsx:148:0-185:2) to extract real-time 
  config from the live Convex subscription. It now displays exact 
  requirements (e.g., "Solve 5 CAPTCHAs") instead of generic descriptions.
- **Specific Penalty Routing:** Enhanced [PenaltySection](cci:1://file://wsl.localhost/Ubuntu-22.04/home/maajith/mono/apps/native/components/ui/modal/EventDetailConditions.tsx:96:0-146:2) to reveal the 
  exact destination of a penalty. If an "Embarrassing Photo" is set to 
  be sent via email, the UI explicitly shows the target recipient's 
  address (e.g., "Photo will be sent to [email]").
- **Visual Consistency:** Standardized all penalty cards to use the 
  Commit "Failure Red" (#FF3B30) regardless of type, providing a clear 
  punitive visual cue.

### 🔧 Technical Improvements
- **Data Directness:** Migrated UI logic to pull from the top-level 
  `penalty` and `penalty_waiver` objects instead of iterating through 
  generic condition arrays.
- **Responsive Status Mapping:** Integrated event status into the 
  condition rows, showing "Applied" for active waivers, "Waived" for 
  successful challenges, and "Failed" for fired penalties.
