refactor(native/enforcement): implement strict background location auditing

Hardens the location permission oracle to require mandatory 'Allow all the 
time' (ACCESS_BACKGROUND_LOCATION) on Android 10+ devices.

Previously, the auditor would return 'true' if only foreground location 
was granted, leading to a UI lie where the dashboard showed a green 
status while the background enforcement engine was actually starving 
of GPS data. 

Changes:
- Added API-level gated check for ACCESS_BACKGROUND_LOCATION.
- Integrated [STRICT ENFORCEMENT PROTOCOL] documentation in native source.
- Validates that both Fine and Background permissions are physically granted 
  before returning a ready status to the frontend.
