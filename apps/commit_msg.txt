feat(backend): implement sovereign instance execution and reusable accountability arming

This commit refactors the verification runner to ensure that every task 
instance — especially manual edits — is treated as a sovereign contract 
that must reach a terminal state, even if the parent task is deleted. 

### 🏗️ Sovereign Instance Execution
- **Decoupled Grading:** Removed early returns in `runVerification` tied 
  to parent task existence. The system now grades an instance based on 
  its own state (Phase 1), ensuring that manual exceptions are processed 
  regardless of the master habit's lifecycle.
- **Heartbeat Protection:** Guarded the `syncTaskSchedule` (Phase 2) 
  with a task existence check. This prevents the heartbeat from reviving 
  deleted habit series while still allowing current in-flight instances 
  to finish their lifecycle.
- **Resolved Chaining Issues:** Fixed a critical edge case where Monday's 
  cleanup would "break the chain" for Wednesday's manual edit. Each 
  instance now executes its contract independently.

### 🛡️ Reusable Accountability Arming

  arming logic into a reusable helper.
- **Support for Pre-emptive Waivers:** Added a `baseTime` parameter. 
  While auto-failures use the window end, this now supports manually 
  starting a waiver session (e.g., via the UI) by passing `Date.now()`.
- **Idempotency & Safety:** Implemented `enforcement_job_id` cancellation 
  within the arming logic to prevent "Double-Firing" during race 
  conditions or overlapping manual triggers.
