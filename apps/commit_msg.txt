style: harden Saffron design system and tokenized UI architecture

Unified the application's visual identity by replacing hardcoded styling with a centralized THEME system.

Key Changes:
- Theme Expansion: Integrated granular spacing, typography scale, and 'pureBlack' OLED tokens in `theme.ts`.
- Background Unification: Replaced all 'bg-black' instances with `THEME.colors.pureBlack` for high-contrast OLED depth.
- Card Refactoring: Migrated `CommitCard`, `VerificationCard`, and `ConditionCard` to use the new Warm Charcoal (`#1F1F1E`) surface token.
- Creation Flow Migration: Fully themed the `(create-commit)` final screen, ensuring all icons, toggles (CustomSwitch), and mini-cards pull from centralized theme tokens.
- Component Hardening: Stripped Tailwind utility classes for colors and radii from core UI components in favor of dynamic theme-driven inline styles.

This ensures a consistent, premium HUD aesthetic across both the main dashboard and the commitment creation pipeline.
