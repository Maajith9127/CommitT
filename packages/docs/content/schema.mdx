  <Tab value="Enums">
    The schema uses several enums to define valid values for certain fields.

    ### Relation Types
    ```typescript
    type RelationType =
      | 'eq'       // Equal to
      | 'neq'      // Not equal to
      | 'gt'       // Greater than
      | 'gte'      // Greater than or equal to
      | 'lt'       // Less than
      | 'lte'      // Less than or equal to
      | 'in'       // Value is in a set
      | 'not_in'   // Value is not in a set
      | 'within'   // Within a geographic area
      | 'outside'  // Outside a geographic area
      | 'exists'   // Value exists
      | 'matches'  // Value matches a pattern
      | 'range';   // Within a numeric range
    ```

    ### Target Types
    ```typescript
    type TargetType =
      | 'number'   // Numeric value
      | 'string'   // String value
      | 'boolean'  // Boolean value
      | 'array'    // Array of values
      | 'range'    // Range of values
      | 'file'     // File reference
      | 'log';     // Log entry
    ```

    ### Permissions
    ```typescript
    type PermissionType =
      | 'manual'       // Requires manual user input
      | 'application'  // Application-level permission
      | 'system'       // System-level access
      | 'network'      // Network access
      | 'location'     // Location access
      | 'media'        // Media access
      | 'external';    // External service access
    ```

    ### Visibility
    ```typescript
    type VisibilityType =
      | 'public'   // Visible to everyone
      | 'private'  // Visible only to assigner and assignee
      | 'shared';  // Visible to assigned users
    ```

    ### Task Status
    ```typescript
    type TaskStatus =
      | 'pending'      // Task created but not started
      | 'proceeding'   // Task is in progress
      | 'proceeded'    // Task completed successfully
      | 'failed';      // Task failed
    ```

  <Tab value="Metrics Table">
    The `metrics` table stores information about available metrics that can be used in task conditions.

    <TypeTable
      type={{
        key: { type: 'string', description: 'Unique identifier for the metric' },
        components: { type: 'string[]', description: 'Array of component names within the metric' },
        name: { type: 'string', description: 'Human-readable name of the metric' },
        description: { type: 'string', description: 'Detailed description of what the metric measures' },
        unit: { type: 'string', description: 'Unit of measurement (e.g., "ms", "geo")' },
        allowed_relations: { type: 'RelationType[]', description: 'Allowed relation types for conditions' },
        allowed_target_types: { type: 'TargetType[]', description: 'Allowed target value types' },
        permissions_required: { type: 'PermissionType[]', description: 'Required permissions to access this metric' },
      }}
    />

    **Indexes:**
    - `by_key`: Indexed on `key` for fast lookups by metric key
    - `by_name`: Indexed on `name` for searching metrics by name
  </Tab>
  <Tab value="Tasks Table">
    The `tasks` table stores task definitions with their conditions and metadata.

    <TypeTable
      type={{
        assigner_id: { type: 'string', description: 'ID of the user who assigned the task' },
        assignee_id: { type: 'string', description: 'ID of the user assigned to the task' },
        title: { type: 'string', description: 'Task title' },
        description: { type: 'string', description: 'Task description' },
        visibility: { type: 'VisibilityType', description: 'Visibility level of the task' },
        conditions: {
          type: 'Condition[]',
          description: 'Array of conditions that must be met',
          properties: {
            metric_key: { type: 'string', description: 'Key of the metric to check' },
            component: { type: 'string?', description: 'Optional component within the metric' },
            relation: { type: 'RelationType', description: 'Relation to apply in the condition' },
            target: {
              type: 'Target',
              description: 'Target value for the condition',
              properties: {
                type: { type: 'TargetType', description: 'Type of the target value' },
                value: { type: 'any', description: 'The target value' },
              },
            },
          },
        },
        status: { type: 'TaskStatus', description: 'Current status of the task' },
        created_at: { type: 'number', description: 'Unix timestamp when the task was created' },
        updated_at: { type: 'number', description: 'Unix timestamp when the task was last updated' },
      }}
    />

    **Indexes:**
    - `by_assignee_id`: Indexed on `assignee_id` for tasks assigned to a user
    - `by_assigner_id`: Indexed on `assigner_id` for tasks created by a user
    - `by_status`: Indexed on `status` for filtering by task status
    - `by_assignee_status`: Composite index on `assignee_id` and `status`
    - `by_assigner_status`: Composite index on `assigner_id` and `status`
    - `by_created_at`: Indexed on `created_at` for chronological sorting
    - `by_updated_at`: Indexed on `updated_at` for recent activity sorting

    **Example Task Data:**
    ```json
    {
      "_id": "task_abc123", // Auto-generated by Convex
      "assigner_id": "user_123", // User who created the task
      "assignee_id": "user_456", // User assigned to complete the task
      "title": "Complete office check-in",
      "description": "Verify presence at office and complete within time window",
      "visibility": "private", // Only visible to assigner and assignee
      "conditions": [
        {
          "metric_key": "location", // References the 'location' metric
          "component": "position", // Check the position component
          "relation": "within", // Must be within the specified area
          "target": {
            "type": "range", // Target is a geographic range
            "value": {
              "lat": 40.7128, // Office latitude
              "lng": -74.0060, // Office longitude
              "radius": 100 // 100 meter radius
            }
          }
        },
        {
          "metric_key": "time", // References the 'time' metric
          "component": "duration", // Check the duration component
          "relation": "lte", // Must be less than or equal to
          "target": {
            "type": "number", // Target is a number
            "value": 3600 // 1 hour in seconds
          }
        }
      ],
      "status": "pending", // Task is waiting to be started
      "created_at": 1640995200000, // Unix timestamp (Jan 1, 2022)
      "updated_at": 1640995200000
    }
    ```
